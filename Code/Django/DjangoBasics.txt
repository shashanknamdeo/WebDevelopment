config/ Folder
--------------
    This is your project brain.

config/__init__.py
    Marks folder as a Python module
    No logic here

config/settings.py
    This controls how Django behaves
    Contains
        Installed apps
        Database config
        Middleware
        Security settings

config/urls.py
    Maps URLs ‚Üí views (APIs)
    Example     :    path('api/users/', ...)
    Think of it as      :   API traffic controller üö¶

        Production servers

Super User / Admin User
-----------------------

Create Super User
    python manage.py createsuperuser

Access Super User
    /admin 

Purpose of Admin User
    Django comes with a built-in admin dashboard (/admin)
    It allows you to manage your database tables via a web interface
    You can:
        Add / edit / delete users
        Check study plans, sessions, feedback
        Test tables and relationships
        Basically, it‚Äôs like a control panel for your backend
Not for app users
    Normal Website / App users will not use this account
    This is only for you / developers / system admins
Benefits
    Quick way to inspect database entries without writing code
    Test that migrations and models are working
    Debug data-related issues early
